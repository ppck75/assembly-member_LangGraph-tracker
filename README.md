# 🕵️‍♂️ assembly-member_LangGraph-tracker

> **LangGraph 기반 대한민국 국회의원 활동 및 입법 프로세스 추적 AI 에이전트**
>
> 본 프로젝트는 디지털 공론장(Digital Public Sphere) 및 감시 저널리즘(Watchdog Journalism) 이론을 고도화된 AI 파이프라인으로 구현한 시스템입니다. 복잡하고 파편화된 국회 데이터를 시민들이 쉽게 이해하고 추적할 수 있도록 돕는 것을 목표로 합니다.    
URL: https://national-assembly.streamlit.app/  
---

## 🌟 주요 기능 (Key Features)

### 1. LangGraph 기반 다단계 입법 맥락 분석 (Multi-stage Reasoning)
- **의제-의원 연계 분석**: 본회의 및 상임위원회 의사일정(`assembly_agenda_cli.py`) 데이터와 특정 의원의 발의 의안을 교차 분석하여, 의원이 어떤 의제에 집중하고 있는지 맥락적으로 추적합니다.
- **감시 저널리즘 기반 논조 요약**: LLM의 단발성 답변 한계를 복잡한 상호작용 그래프(State Graph)로 극복하여, 법안의 핵심 쟁점과 찬반 입장을 객관적으로 비교·요약합니다.

### 2. 8단계 청원 처리 상태 분류 및 데이터 파이프라인
- **정밀 데이터 라벨링**: 국회 Open API에서 파편화되어 제공되는 국민동의청원 데이터를 수집하여, 계류·심사·채택·폐기 등 복잡한 과정을 본 프로젝트만의 **'8단계 표준 처리 상태 파이프라인'**(`petition_selection_cli.py`)으로 자동 분류합니다.
- **병목 구간 분석**: 각 청원이 어느 심사 단계에서 오랫동안 정체되어 있는지, 심사 지연 기간을 계산하고 데이터화하여 입법부의 태만이나 병목 구간을 분석합니다.

### 3. 직관적인 의정 활동 시각화 컨텍스트 (Visual Context)
- **입법 활동 타임라인(Timeline) 시각화**: 특정 국회의원이나 법안의 발의부터 상임위 상정, 본회의 표결, 최종 통과에 이르는 전 과정을 시각적인 타임라인 형태로 재구성하여 제공합니다.
- **청원 진척도 스테이터스 바(Status Bar)**: 8단계 분류 시스템과 연동되어, 대중이 특정 청원의 현재 심사 위치와 남은 절차를 한눈에 직관적으로 파악할 수 있는 시각적 인디케이터 컨텍스트를 생성합니다.
- **이슈 키워드 맵 데이터 구조화**: 실시간 국회 의사일정 데이터에서 추출한 핵심 키워드의 빈도와 중요도를 분석하여, 현재 국회 내의 주요 쟁점을 한눈에 보여줄 수 있는 시각화 인터페이스용 정형 데이터를 생성합니다.

## 🏗️ 시스템 아키텍처 및 워크플로우 (Architecture)

본 시스템은 데이터 수집/전처리 레이어와 LangGraph 기반의 추론/에이전트 레이어로 분리되어 상호작용합니다.

```text
[열린국회정보 Open API]
         │
         ▼ (실시간 수집 및 분류)
┌────────────────────────────────────────┐
│  • petition_selection_cli.py (청원)     │ ──► 8단계 상태 분류 시스템 적용
│  • assembly_agenda_cli.py   (의사일정)  │
└────────────────────────────────────────┘
         │
         ▼ (구조화된 데이터 적재)
┌────────────────────────────────────────┐
│     의원_활동_추적_랭그래프_워크플로우.ipynb  │ ──► LangGraph State Graph 기반 
│     (LangGraph Agent Workflow Layer)   │     LLM 맥락 요약 및 답변 생성
└────────────────────────────────────────┘
